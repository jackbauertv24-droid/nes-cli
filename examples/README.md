# NES CLI Examples

This folder contains example outputs generated by nes-cli using Super Mario Bros. 3 (USA) ROM.

## Files

| File | Description | Generated Command |
|------|-------------|-------------------|
| `smb3_intro.gif` | 10-second GIF animation (title screen) | `record:gif:10s` |
| `smb3_audio.wav` | 10-second audio recording | `audio:10s` |
| `smb3_game.png` | Screenshot at frame 600 | `screenshot` |
| `smb3_gameplay.png` | Screenshot during gameplay (after START) | `screenshot` |
| `smb3_ascii.txt` | ASCII art screenshot | `screenshot --format ascii` |
| `smb3_ansi.ansi` | ANSI colored screenshot | `screenshot --format ansi` |

## Sprites

### Title Screen Sprites (`sprites/`)

```
sprites/
├── chr/
│   ├── spritesheet_gray.png         (256 tiles, grayscale)
│   ├── spritesheet_palette0.png      (256 tiles, game palette 1)
│   ├── spritesheet_palette1.png      (256 tiles, game palette 2)
│   ├── spritesheet_palette2.png      (256 tiles, game palette 3)
│   └── spritesheet_palette3.png      (256 tiles, game palette 4)
├── oam.json                           (metadata for 64 sprites)
└── oam/
    └── spritesheet.png               (visible sprites grid)
```

### Gameplay Sprites (`sprites_gameplay/`)

Same structure as above, but captured during gameplay (different palettes).

## Output Specifications

- **GIF**: 256×240 pixels, ~60fps, approximately 300-500KB for 10 seconds
- **WAV**: 44100Hz sample rate, 16-bit PCM, mono, approximately 480KB for 10 seconds  
- **PNG**: 256×240 pixels, RGBA, approximately 5-20KB per frame
- **CHR spritesheets**: 8×32 grid of 8×8 tiles (256 total)
- **OAM spritesheet**: Grid of visible sprites currently on screen

## Note

These examples were generated using a legally obtained ROM for demonstration purposes. Always respect copyright laws when using nes-cli.
